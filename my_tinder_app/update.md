1. 快取機制 (Caching) - 最有效
原理： 不要每次都問 Google，先問自己的資料庫。
作法： 當有人在「台大校門口」查過餐廳，你就把查到的這 20 間餐廳存進 Firestore (標記座標)。
省錢流程： 下次有別人在同一個地點查詢，先檢查 Firestore 有沒有資料。
有 -> 讀 Firestore (成本趨近 $0)。
沒有 -> 呼叫 Google Maps API (成本 $1)。
效果： 隨著用戶變多，你的 API 呼叫次數反而會下降，因為大家都查過一樣的地方了。
2. 改為「手動更新」
原理： 只有使用者真的想找新地點時才花錢。
作法： 把 _startLocationUpdates 裡的自動呼叫拿掉。改成在畫面上放一顆「搜尋此區域」的按鈕，使用者按下後才呼叫 API。
3. 設定 API 每日額度 (Quotas)
原理： 設定停損點。
作法： 去 Google Cloud Console -> Google Maps Platform -> Quotas。
設定 Places API 的每日上限為 200 次。
這樣就算程式寫錯或是被惡意攻擊，當天頂多 App 壞掉 (顯示無法連線)，但不會讓你的信用卡刷爆。
